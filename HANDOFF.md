## Key Current Decisions

Load-bearing invariants for anyone touching Orb behavior. Full operating rules live in **AGENTS.md**; UI patterns in **docs/ui-catalog.md**; conversation behavior in **eval cases** + `lib/orb-contract.ts`.

- **Name-first identifiers.** Project **NAME** is the identifier everywhere users/model interact. Project **code** is internal-only, auto-generated, immutable, and exists solely to prefix todo codes (`ORB-73`) — never a second user-facing label, never user-editable. References resolve name → exact code → fuzzy/partial name (`resolveProjectByReference`). `switch_project`, `update_project`, `delete_project` all take **name**, server resolves. (The old `switch_project`-uses-code inconsistency is **reconciled** as of 2026-07-12.)
- **Todo identity model** (decided 2026-07-14 with Stan; full reasoning on **ORB-337**). Three identifiers, three jobs — stated explicitly because it reads as obvious and gets silently violated when unwritten:
  - **`id` (uuid)** — unique and **permanent**, never changes. The true identity. Anything needing a durable reference stores this.
  - **`code`** (project code + `todo_number`, e.g. `ORB-336`) — unique at any moment, unique **forever** under never-recycle, but **not permanent for a given todo**: it changes on move. It is the current **address**, not the identity.
  - **`title`** — **not unique, and deliberately must not be.** A human search key. Do not add a uniqueness constraint; the resolver fails closed on ambiguity (exact → ranked → refuse). Already the stated principle in `lib/orb-mutations.ts` ("Identity is a resolved row id, never a free-text name") and Realtime's `resolveTodoReference`.
  - **Move renumbers** (decided yes): a moved todo takes the destination's next available number, since its old number may already be taken there. New address, same identity.
  - **The two rules need each other:** renumber-on-move is only safe *because* codes are never recycled. Together, an old code either still names the same todo or names nothing — never a *different* todo. That converts a stale reference from **silently wrong** into **honestly dangling**, which is the whole point of never-recycle.
  - **Knowledge Repo needs no schema change here:** `origin_todo_id` is already the uuid and survives a move (FK `ON DELETE SET NULL`; Phase 1's close writes it). A code in prose is a snapshot — accurate as of writing, at worst dangling, never wrong. `audit_log` records `todo_move` before/after codes against the uuid, so retired code → uuid → current code stays reconstructable.
- **Structural mutation gate** (not prompt-only). CRUD tools are held server-side: the model calls the tool immediately, the server persists a proposal, and shared server predicates evaluate the actual current utterance. Upfront permission executes without a duplicate ask; otherwise only a bare affirmation confirms. `confirm_mutation` is unavailable to the serial model unless those conditions hold, and the handler rechecks defensively, so mixed complaints/discussion cannot self-confirm through the model.
- **Identifier provenance.** Task/project codes may only be used if they were actually seen this conversation (backlog, tool result, or the user's words) — never constructed by pattern or remembered across a cleared session. Enforced at the prompt layer + a server-side gate that rejects mutations targeting unseen codes.
- **Browser support.** Safari, Chrome, and Edge are the required representative browser families. Firefox is temporarily experimental and tracked separately by ORB-330 after repeated multi-turn microphone failure. Canonical policy: `docs/browser-support-policy.md`.
- **Voice — Realtime IS the destination (decided 2026-07-14 by Stan).** Realtime/WebRTC becomes the main voice control; the serial path is not the long-term answer. This is settled direction, not an open question — but the **flip lands after typed parity** (Phases 2–5), the ambient-false-turn work, supported-browser acceptance, and a green full Tier 1. Flipping earlier would strand users without project mutations, knowledge, tickets, audit, and navigation. Until then production still defaults to the serial path, and the Realtime operator stays dev/allowlist-gated. Do not restore pseudo-realtime sentence-level `speakStreaming()`. Add neither phrase-specific patches nor canned responses: natural language stays fluid, while deterministic state, routing, authorization, confirmation, idempotency, cancellation, and verified outcomes enforce correctness.
