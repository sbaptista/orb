## 5. Fix .env.local not being read

**Task:** Diagnose why dotenv loaded 0 variables from `.env.local`.

**Actions:**
- `ls -la` revealed the file was named `.env.local ` (trailing space in filename)
- Renamed it: `mv ".env.local " ".env.local"`
- Re-ran the script — new error: `__dirname is not defined in ES module scope`
- Node was treating the file as an ES module (due to `import` syntax); `__dirname` is not available in ESM
- Attempted fix using `import.meta.url` — blocked by ts-node compiling as CommonJS
- Final fix: replaced `path.join(__dirname, '..', 'scripts', ...)` with `path.join(process.cwd(), 'scripts', ...)`, which resolves correctly when the script is run from the project root
